                Laptop Improvisation with Live Instruments

                Challenges

                • Exchange with instrumentalists
                • Spontaneous reaction
                • Quick response
                • Sensitive and reflective to musiciansâ€™ contributions

                Live Coding

                • Live coding already provides the modularity needed for laptop improvisation.
                • In practice it often involves typing relatively long pieces of code.
                • Good for process music - slowly evolving music, drone, dance music, etc.
                • Difficult to achive spontaneous reaction, particularly when interacting with live instrumentalists.

                Modularity

                • Laptop improviser is able to choose depending on musical circumstances at a specific moment
                • SuperCollider
                • Live Coding

                Immediacy / Exchange

                • Improvisation with live instruments often requires fast exchange
                • In practice, live coding frequently involves typing long pieces of code
                • Not desirable for immediate response to instruments

                Live coding environment

                Laptop improvisation: dialog between preconceived and spontaneous decisions

                Considerations:

                • Short and idiosyncratic commands
                • Less typing is better!
                • Little typing / quick audible results
                • Easily extendible
                • Few/No GUI
                • No Complications (focus on music)
                • Fixed setup for external controllers
                • Dynamic and static controls
                • Individual and global controls
                • Dynamic buffer allocation
                • Save and load presets and environment settings at different levels
                • Automation of many parameters
                • Combining as many SC features as possible
                • Accessing these features should be very easy

                SC Features:

                • Synthesis, Buffer Manipulation, Patterns, Live Signal Processing
                • Live coding (Just-in Time Library)
                • Algorithmic Composition
                • Music Information Retrieval (SCMIR)
                • Machine Listening

                Future developments:

                • Planes: Analysis, Generative and Deterministic Planes
                • Space: More Spatialisation
                • Triggers: More ways of setting events off
                • More complex audio routing
                • More synths, effects, sound files, etc.
                • More SCMIR
                • Machine Learning
                • Multimodal Interaction: Adding Computer Vision
